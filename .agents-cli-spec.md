# Agent Spec — SEO + AEO Multi-Agent System

## Overview
A multi-agent system (Google ADK) that takes a **niche + target website URL** and drives a
full SEO **and** AEO/GEO (Answer/Generative Engine Optimization) program end-to-end:

1. **Analyze** the site + competitors (technical, on-page, keyword, backlink, SERP/AEO).
2. **Conclude** — synthesize grounded findings (every claim traces to real tool data).
3. **Strategize** — topic-cluster map, prioritized roadmap, content briefs.
4. **Implement** — apply approved changes via a CMS adapter (human-gated).
5. **Document** — a change log of exactly what was done and why.
6. **Score** — a deterministic before/after Health Score with per-category deltas.

It runs as **three checkpointed phases** (human approval between each) so nothing runs
unbounded and the agent is always boxed into one lane:

- **Phase 1 — DIAGNOSE & STRATEGIZE** (read-only): analyze → conclude → map → strategize → baseline score.
- **Phase 2 — IMPLEMENT** (gated writes): execute approved roadmap items → change log.
- **Phase 3 — VERIFY & SCORE**: re-crawl, confirm changes live, recompute score, report improvement.

## Design principles (anti-hallucination)
Hallucination is prevented **structurally**, not by instruction alone:
1. **Instruction contracts** — every agent has explicit ROLE / MUST / MUST NOT / IF-UNSURE.
2. **Typed outputs** — Pydantic `output_schema` on terminal/reasoning agents.
3. **Grounding discipline** — findings must cite the tool field they came from; missing data → `unavailable`, never estimated.
4. **Runtime callbacks** — read-only enforcement per phase, quota/rate-limit tracking, publish approval gate.
5. **Human checkpoints** — between phases + confirmation before every live write.
6. **Scope isolation** — leaf agents `disallow_transfer_to_peers`; `google_search` isolated in its own agent.

## Three knowledge layers (kept separate on purpose)
- **Deterministic rules** (`config.py`): exact thresholds & checks (sitemap 50k/50MB, robots.txt 500KiB,
  CWV bands LCP 2.5s / INP 200ms / CLS 0.1, title ~575px, crawlable-link rules, API quotas). NOT RAG.
- **Best-practice knowledge (RAG)** (`knowledge/` + `knowledge_base/`): curated Google Search Central docs,
  AEO/GEO playbook, Princeton citation levers, our research. Local Chroma-free numpy vector store for the
  prototype; upgrade path to Vertex AI Search on deploy.
- **Live data** (Semrush MCP + Google APIs): competitor keywords, volumes, backlinks, rankings, CWV field data.

## Tools Required
- **crawler_tools** — fetch + parse HTML; **link/anchor auditor** implementing Google's crawlable-links rules
  (flags `routerLink`/`span href`/`onclick`-only/`javascript:` links, generic/stuffed anchors, missing alt/title,
  chained links, orphan/over-linked pages, missing `rel=sponsored/ugc`); technical basics (robots, sitemap,
  canonical, meta-robots, title/meta length, H1 count, JSON-LD presence). *Works out of the box.*
- **pagespeed_crux_tools** — PageSpeed Insights + CrUX field CWV. *Needs `PAGESPEED_API_KEY` (optional).*
- **gsc_tools** — Search Analytics + URL Inspection. *Needs a GSC service account.*
- **ga4_tools** — GA4 Data API organic metrics. *Needs GA4 creds.*
- **semrush_mcp** — `McpToolset` to a Semrush MCP server. *Needs `SEMRUSH_MCP_URL` + key.*
- **scoring_tools** — deterministic Health Score + before/after diff. *Pure code, no LLM.*
- **knowledge_retrieval** — local RAG over the curated corpus. *Active after `ingest`.*
- **cms_publish_tools** — abstract `Publisher` + stub; Webflow/Shopify adapters plug in. *Gated.*

## Constraints & Safety Rules
- **No fabrication.** Findings/metrics/URLs must come from tool output; otherwise report `unavailable`.
- **Read-only Phase 1.** Write/publish tools are blocked unless `state["phase"] == "implement"`.
- **Publish gate.** CMS writes blocked unless `state["publish_approved"] is True` (human sets it).
- **Indexing is verify-only.** Google Indexing API is JobPosting/Broadcast only; we verify via URL Inspection + sitemaps.
- **Reranking is measure-only.** Not controllable; optimize genuine engagement + E-E-A-T proxies.
- **Respect API quotas** (URL Inspection 2,000/day/site is the tightest) — tracked in state.
- **Third-party authority scores** (DR/DA/AS) are relative proxies, never ground truth.

## Success Criteria
- Phase 1 produces a grounded Findings report + prioritized Roadmap + baseline Scorecard with sources.
- Phase 2 applies only approved items and logs each change.
- Phase 3 recomputes the score and reports per-category improvement tied to specific actions.
- No finding without a traceable source; no live write without approval.
