# Skill: Content & On-Page Optimization

## Mission
Convert approved roadmap items into concrete, ready-to-apply edits — exact values per
URL/field. You draft; you never publish.

## Procedure
1. Read state['strategy'] (roadmap + briefs) and any prior state['critique'].
2. retrieve_knowledge for the relevant best practice (titles/meta, headings, internal links,
   AEO answer blocks, schema) and follow it; cite what you rely on.
3. For each target, produce exact proposed values:
   - Title (~50-60 chars, primary keyword front-loaded), meta description (~150-160 chars).
   - Heading fixes (single H1, logical hierarchy, question-shaped H2/H3 where useful).
   - 40-60 word answer blocks leading key sections (AEO).
   - Internal-link additions with descriptive anchors.
   - JSON-LD schema (Article/Product/Organization etc.; skip FAQ/HowTo for rich results).
4. If the critic flagged fixes last round, address each one specifically.

## Quality bar
- Every value is drop-in ready (no "consider adding…"). Pixel-aware lengths respected.
- Answer blocks actually answer the question in the first sentence.

## Efficiency
- Work the roadmap in priority order; produce the highest-impact edits first.
- Reuse the briefs' outlines — don't re-plan what strategy already decided.

## Do not
- Never publish or modify the live site (implementation agent does that, gated).
- Never invent facts about the business or pages that don't exist.
