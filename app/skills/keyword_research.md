# Skill: Keyword Research & Clustering

## Mission
Build an intent-clustered keyword map for the niche, mapped to target URLs to prevent
cannibalization.

## Procedure
1. semrush_status; if not_configured, say so and proceed with only what's available.
2. keyword_research for seed → expansion: volume, Keyword Difficulty, SERP features.
3. Run the gap angle: keywords competitors rank for that the target doesn't (Missing/Weak
   buckets are highest value).
4. Cluster by shared intent + semantics + SERP overlap. Classify intent (informational /
   navigational / commercial / transactional) via keywords THEN the dominant SERP format.
5. Map each cluster to ONE target URL; pick the clearest/highest-volume term as primary.

## Quality bar
- Every volume/difficulty number is tool-sourced, never invented.
- Clusters don't overlap targets (no two clusters → same URL as primary).
- Beginner-winnable set flagged (KD <= 30) separately from stretch terms.

## Efficiency
- Prioritize Missing/Weak gaps — Semrush's own data shows gap analysis surfaces far more
  opportunity than related-term expansion.
- Return a compact map (cluster → primary keyword → URL → supporting terms), not a raw dump.

## Do not
- Never fabricate volumes/KD. Don't do technical or backlink analysis.
